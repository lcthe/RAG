## 一、今天做了什么

实现了完整的检索系统：向量检索 + BM25 + 混合检索 + 重排序。

- VectorStore：基于 FAISS，支持 Flat/IVF/HNSW 三种索引
- BM25Retriever：关键词检索，中英文分词
- HybridRetriever：向量+BM25 加权融合
- Reranker：基于关键词重排

## 二、向量数据库选型

### 2.1 ANN 算法对比

| 算法 | 原理 | 速度 | 召回 | 场景 |
|------|------|------|------|------|
| **Flat** | 暴力搜索，全量遍历 | O(n) | 最高 | 数据量<1万 |
| **IVF** | 倒排索引，分桶搜索 | O(sqrt(n)) | 中 | 1万-100万 |
| **HNSW** | 分层小世界，图索引 | O(log n) | 高 | 大+高召回，首推 |
| **PQ** | 乘积量化，压缩向量 | 极高 | 中低 | 内存敏感场景 |

### 2.2 主流向量库对比

| 向量库 | 语言 | 特点 | 适合场景 |
|--------|------|------|----------|
| **FAISS** | C++/Python | 本地库，最快，内存索引 | 学习/小规模生产 |
| **Chroma** | Python | 轻量嵌入式，自动持久化 | 快速原型 |
| **Qdrant** | Rust | 高性能，支持过滤 | 中小型生产 |
| **Milvus** | Go | 分布式，海量数据 | 大规模生产 |

## 三、核心接口设计

### 3.1 VectorStore

`
add(vectors, texts, metadatas) 
    → FAISS 索引 + 文本存储
    
search(query_vector, top_k) 
    → List[SearchResult]
    
save(path) / load(path) 
    → 持久化到磁盘
`

### 3.2 BM25Retriever

`
add_texts(texts, metadatas)
    → 分词 + BM25 索引
    
search(query_text, top_k)
    → List[SearchResult]
`

### 3.3 HybridRetriever

`
search(query_vector, query_text, top_k)
    → 1. 向量检索 Top-K*2
    → 2. BM25 检索 Top-K*2
    → 3. 分数归一化
    → 4. 加权融合 (0.7向量 + 0.3BM25)
    → 5. 排序取 Top-K
`

### 3.4 完整流程

`
添加知识库:
  文档 → 分段 (Day2) → Embedding (Day3) → FAISS + BM25 双索引

用户查询:
  问题 → Embedding → 向量检索 → 混合检索
                    → 关键词检索 → 
                              ↓
                          重排序
                              ↓
                          Top-K 结果
`

## 四、检索测试结果

### 4.1 纯向量 vs 混合检索对比

| 查询 | 纯向量最佳匹配 | 混合匹配 | 混合提升 |
|------|---------------|---------|---------|
| 音箱价格 | 0.548 | 0.623 | ✅ BM25 加持 |
| 保修期 | 0.541 | 0.490 | ⚠️ BM25 无匹配 |
| 音频规格 | 0.306 | 1.032 | ✅ 关键词强匹配 |

### 4.2 核心发现

- 向量检索：语义泛化好，但分数差异小
- BM25：关键词精确，但不懂语义
- 混合：两者互补，但需调整权重
- **最佳实践**：向量 0.7 + BM25 0.3，先粗筛再精排

## 五、距离度量选型

| 度量 | 公式 | 范围 | 最适合 |
|------|------|------|--------|
| **余弦相似度** | cos(A,B) | [-1, 1] | 文本/语义 |
| 欧氏距离 | \|A-B\| | [0, +∞) | 图像/数值 |
| 点积 | A·B | (-∞, +∞) | ML 内积 |

**FAISS 实现：** IndexFlatIP（内积）+ 
ormalize_L2（归一化）= 余弦相似度。

## 六、专业术语速查

| 术语 | 含义 | 用在哪 |
|------|------|--------|
| **ANN** | 近似最近邻搜索，找最相似向量 | "用 ANN 算法加速检索" |
| **HNSW** | 分层小世界图，最常用的 ANN | "百万数据推荐 HNSW" |
| **IVF** | 倒排索引，分桶搜索 | "IVF 适合中等规模" |
| **PQ** | 乘积量化，压缩向量减少内存 | "PQ 内存省 90%" |
| **BM25** | 传统关键词检索算法 | "BM25 弥补向量检索的弱点" |
| **Hybrid Search** | 混合检索=向量+关键词 | "混合检索效果最好" |
| **Rerank** | 重排序，对粗筛结果精排 | "先用向量粗筛，再 rerank" |
| **Cross-encoder** | 单塔模型，query+doc 一起编码 | "cross-encoder 精度高但慢" |
| **Dual Encoder** | 双塔模型，query 和 doc 分别编码 | "向量检索就是双塔" |
| **Recall** | 召回率，相关结果被命中的比例 | "召回率 95%" |
| **Precision** | 精确率，命中结果中是相关的比例 | "精确率 80%" |
| **Top-K** | 返回最相关的 K 个结果 | "Top-5 检索" |
